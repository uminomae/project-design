# docs/ — 管理ドキュメントハブ

project-design プロジェクトの管理書類。CLAUDE.md が全ての正本。

## エントリーポイント

| ファイル | 内容 | 読むタイミング |
|---------|------|--------------|
| `CLAUDE.md` | 権限・原則・セッション手順 | 常時（自動読込） |
| `docs/README.md` | 管理ドキュメント一覧 | 状況把握時 |
| `docs/PROJECT.md` | プロジェクト憲章 | 方針確認時 |
| `.cache/session/state.md` | セッション状態 | セッション開始時 |

## ファイル一覧

| ファイル | 内容 |
|---------|------|
| `PROJECT.md` | プロジェクト憲章（目的・スコープ・設計方針） |

## ディレクトリ構造

```
project-design/
├── CLAUDE.md              # 権限・原則の正本
├── docs/                  # 管理ドキュメント
├── knowledge/evidence/    # 調査データ蓄積
├── transform/             # 変換ルール・テンプレート
├── publish/               # 公開出力（正本は pjdhiro）
├── src/                   # Web UI（将来）
├── assets/                # 静的資産
├── scripts/               # ビルド・同期スクリプト
└── .cache/                # セッション状態（git 除外）
    ├── session/           # state.md, ログ, ハンドオフ
    ├── inbox/             # 受信指示
    ├── outbox/            # 送出成果物
    └── active/            # 進行中タスク
```

## 設計原則

creation-space / kesson-driven-thinking のパターンを参考に、PD 固有の文脈で独立運営する。

- **L1 認知的連続性**: state.md, ログ, ハンドオフでセッション間を橋渡し
- **L2 権限境界の明示**: 委任レベル表で人間/AI の境界を宣言
- **L3 コンテキスト圧縮**: 必要な情報を必要な時に
- **L4 構造的品質保証**: ルールで担保、記憶に頼らない
- **S2 保持が解決に先立つ**: 不快と混乱は保持論点として残す
