# プロジェクト憲章 — project-design

## 目的

プロジェクトデザイン論を探索・構造化し、独立した理論体系として管理する。

## スコープ

### 含むもの

- プロジェクトデザイン論の探索・調査
- 調査データの蓄積と構造化（knowledge/evidence/）
- 公開用コンテンツの変換ルール管理（transform/）
- 関連理論（欠損駆動思考、創造モデル）との接続

### 含まないもの

- 公開 MD の正本管理（→ pjdhiro）
- 欠損駆動思考の詳細な理論構築（→ kesson-driven-thinking）
- 創造モデルの詳細な探索（→ creation-space）

## コンテンツ領域

| 領域 | 概要 |
|---|---|
| PD 総論 | 射程・領域・Doing/Being・Love駆動開発 |
| 対話 | 創造、受容、場と時代、概念共有、精神、PD、2025 |
| 思考法 | 欠損駆動思考、BI思考 |
| 感情処理 | 瞑想、情動伝染、感情メタ認知、心理的境界、価値、受容、総論 |
| 用語 | プロジェクトの定義、その他 |

## 設計方針

- **公開 MD の正本は pjdhiro**: creation-space と同じパターン
- **knowledge → transform → publish**: 探索データを変換して公開
- **保持論点を急いで解くな**: 不確実なものは保持論点として残す
