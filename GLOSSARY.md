# GLOSSARY — project-design 用語集

本ファイルはリポジトリ運用で使われる用語を定義する。
PD 理論用語の正本は pjdhiro `_pages/pd/word/` にある。

## PD 理論用語（概要のみ）

| 用語 | 定義 |
|------|------|
| プロジェクト | 「やること（Doing）」＋「起きていること（Being）」を含む出来事 |
| Doing | 計画・実行・管理など意図的行為の側面 |
| Being | 感情・関係・場の力学など「起きている」側面 |
| Love 駆動開発 | 関係・感情・意図が駆動する局面に名前を与える試み |
| 欠損駆動思考 | 欠損（不在・不足・違和感）を起点に思考を展開する方法 |
| BI 思考 | Being 起点の思考法 |
| 保持論点 | 不快や混乱を急いで解かず保持する未解決の問い |

## リポジトリ運用用語

| 用語 | 定義 |
|------|------|
| state.md | セッション状態を記録するファイル（`.cache/session/state.md`） |
| inbox | CLI への指示・依頼を置くディレクトリ（`.cache/inbox/`） |
| outbox | CLI からの報告を置くディレクトリ（`.cache/outbox/`） |
| active | 進行中タスクの計画を置くディレクトリ（`.cache/active/`） |
| ハンドオフ | DT App ↔ CLI 間の引き継ぎ文書 |
| DIC | 成果物整合性チェック（Deliverable Integrity Check） |
| pjdhiro | 対話する人間の固有名詞 |

## タグ

| タグ | 意味 |
|------|------|
| [P] | 確立事実（Proven） |
| [M] | 比喩的解釈（Metaphorical） |
| [S] | 推測仮説（Speculative） |

## 委任レベル

| レベル | 説明 |
|--------|------|
| 自律実行 | pjdhiro の承認なしに実行できる操作 |
| 確認後実行 | pjdhiro に確認してから実行する操作 |
| pjdhiro 専権 | pjdhiro のみが判断・実行できる操作 |

各レベルの具体例は `CLAUDE.md` §委任レベル を参照。
