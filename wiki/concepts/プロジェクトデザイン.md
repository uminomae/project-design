# プロジェクトデザイン

やること（Doing）＋起きていること（Being）を含む出来事の設計論。[[欠損駆動思考]]を中核概念とし、予測符号化の神経科学、精神分析（Bion, Klein）の臨床知、日本思想を横断する理論体系。

## 概要

プロジェクトデザインは、「プロジェクト」を単なるタスクの集合体としてではなく、Doing（やること）と Being（起きていること）の両面を含む「出来事」として捉える設計論である。

従来の[[デザイン思考]]が人間中心の問題解決アプローチとして Doing の側面に焦点を当てるのに対し、プロジェクトデザインは Being（感情、関係、意図）の側面を理論的に位置づけることを目指す。

## 3つの展開場所

プロジェクトデザイン論は、3つのリポジトリで展開されている。

### 理論（kesson-driven-thinking → kesson-space）

理論の正本。対話と先行研究を通じて精緻化された概念体系を管理する。D1-D4 のコア定義、[[4層モデル]]、創造の5段階モデルなどが含まれる。

### 体験（kesson-space）

理論を3D空間として追体験できる場所。意識の内側で起きていることを、視覚と身体感覚で感じ取るための空間。Three.js と GLSL シェーダーによる実装。

### 言語（pjdhiro）

ブログ記事とPDF文書。理論を言語で伝える場所。一般版・設計者版・学術版の3つの読者別バージョンがある。

## コア概念（D1-D4）

プロジェクトデザインの理論的基盤は、4つのコア定義から構成される。

| ID | 概念 | 定義 |
|----|------|------|
| D1 | [[欠損駆動思考]] | 棄却される誤差を、問いとして拾う態度 |
| D2 | [[欠損]] | 予想と現実の誤差を、意識が「欠け」として捉えた主観的経験 |
| D3 | [[抱持]] | 反射的に処理せず、誤差を問いとして保持する機能 |
| D4 | [[情動の構成]] | 欠損が生存軸と信頼軸で評価され、情動として構成されるプロセス |

## デザイン思考との関係

プロジェクトデザインは[[デザイン思考]]を否定するのではなく、その射程を拡張する。

| 側面 | デザイン思考 | プロジェクトデザイン |
|------|------------|-------------------|
| 焦点 | Doing（問題解決） | Doing + Being（出来事） |
| 不確実性への対処 | 反復プロトタイピング | [[抱持]]（問いとして保持） |
| 感情の扱い | 共感（empathy） | [[情動の構成]]の理解 |
| 間主観性 | ユーザー中心 | [[間主観性]]（相互了解） |
| 推論形式 | アブダクション | [[欠損駆動思考]]（問いの発見） |

## Love 駆動開発

[[Love 駆動開発]]は、プロジェクトデザインの実践論。関係・感情・意図が駆動する局面に名前を与える試みとして位置づけられる。

## 横断する知の領域

プロジェクトデザインは以下の知の領域を横断する。

| 領域 | 接続する概念 |
|------|------------|
| 神経科学 | 予測符号化、[[内受容感覚]]、[[4層モデル]] |
| 精神分析 | Bion のα機能、Klein の対象関係論、Meltzer の美的葛藤 |
| 発達心理学 | [[間主観性]]、[[愛着理論]]、社会的アロスタシス |
| 現象学 | 一人称記述、身体性 |
| 日本思想 | 「未発の中」（中庸）、「秘すれば花」（世阿弥）、「忍ぶ恋」（葉隠） |
| 設計論 | [[デザイン思考]]、[[ウィキッド・プロブレム]]、アブダクション |

## 情報の流れ

```
理論（kesson-space）
|
+-- 素材（knowledge/）
|   +-- 構造モデル（D1-D4, 4層, 5段階）
|   +-- 本文（意識の作動構造, 創造の構造とプロセス）
|   +-- 論拠DB（先行研究との対比）
|
+-- 変換（transform/）
|   +-- 読者別ルール → PDF, ブログ記事
|   +-- 体験向けルール → 3D空間
|
+-- 出力先
    +-- kesson-space → 追体験する
    +-- pjdhiro → 読む
```
