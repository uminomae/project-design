# Evidence 制約ルール v1.0

**目的**: 公開物の生成時に LLM が evidence にない理論・主張を一般知識から挿入することを防止する
**適用対象**: 全 repo の公開物生成
**正本**: project-design

---

## 1. Evidence-Only 原則

公開物に記載する先行研究者名・理論名は、各 repo の evidence に記録されているもののみ使用可能。

- evidence に存在しない研究者・理論は、一般知識として正しくても記載禁止
- evidence のエントリで裏付けを追跡できること
- 迷った場合は記載しない。記載しないことによる品質低下より、逸脱による信頼性低下のほうが深刻

## 2. 禁止パターン一覧

### 2-1: Evidence にない理論・人物の言及
evidence に記載されていない理論・人物を公開物に登場させてはならない。

### 2-2: 因果逆転
調査がモデルの根拠・出典・起源であるかのような記述は禁止。

禁止パターン:
- 「調査から見出された」「調査により発見された」「調査に基づいて構築された」
- 「investigation revealed」「discovered through research」「derived from survey」

正しい因果は各 repo で定義する。

### 2-3: 解釈の事実的断定
evidence の解釈・読み取り層の内容を、あたかも確立された事実であるかのように記述してはならない。

### 2-4: Evidence にない定量的主張
「最も多い」「N件で確認された」等の定量的主張は、evidence の集計に基づくこと。
LLM の印象や推測で数値・頻度を記述しない。

### 2-5: Evidence にない因果断定
「~が質を決める」「~を保証する」等の因果断定は、evidence が因果を確認していない限り禁止。
代替表現: 「~と対応している」「~と構造的に関連する」

## 3. Temperature ルール — Evidence の層に応じた記述温度

| Evidence 層 | 記述温度 | 使用可能な表現 | 禁止される表現 |
|------------|---------|--------------|--------------|
| 一次文献に基づく事実 | 事実 | 「~である」「~と述べている」 | -- |
| 構造的類似の読み取り | 解釈 | 「~と構造的に対応する」「~として読める」「~と捉えることができる」 | 「~である」「~と証明されている」 |
| Evidence なし | 記述禁止 | -- | いかなる表現も禁止 |

## 4. 検証方法

### 機械チェック
各 repo が検証スクリプトを用意する。

### 目視チェック
各 repo の quality-test に従う。
