## 軸ごとの評価

### 1. 構造化

**ChatGPT: A** — 表4つ + Mermaid 図1つ。5セクションが「出力形式 A-E」のラベルで構成意図ごと明示。見出し・表・図を自発的に組み合わせた構造化力が最も高い。

**Pro: B+** — 5セクション (A-E) で論理的に整理されているが、表・図はゼロ。全て散文。領域別一覧が段落列挙のため一覧性が低い。

**Codex: B+** — 8セクション、H2/H3 の2階層で一貫。箇条書き中心で読みやすい。表・図はなし。

### 2. 網羅性

**ChatGPT: A** — 15領域を横断的にカバー。各領域を「何が価値か / どう測るか / 典型的問い」の統一フォーマットで整理。領域横断の論点も7軸で比較。

**Pro: A-** — 13領域 + 横断論点8項目。ChatGPT とほぼ同等の射程だが、表形式でないため領域間比較がしにくい。

**Codex: B** — 5分類 + 英語5語で概観を整理。心理学（Schwartz 価値理論）、社会学（Bourdieu 文化資本）、人類学（Graeber 価値論）、美学の扱いが薄い。概観としては十分だが、深度を犠牲にした簡潔さとのトレードオフ。

### 3. 内容正確性

**3レポートとも A-** — 主張と既知の学術概念（intrinsic/instrumental value, VSD, プロスペクト理論, ACT の values 等）の整合性は高い。明確な事実誤認は検出されなかった。ChatGPT の Cambridge Dictionary の記述は 403 で直接確認できなかったが、同辞書が value/values の区別を扱うことは周知。

### 4. 検証可能性

**Codex: A** — 出典3件（SEP, Britannica, コトバンク）に URL 付き。3件とも実在確認済み（V1 通過）。出典数は少ないが、信頼性の高い百科事典的ソースに集中。

**ChatGPT: C** — `citeturnXsearchY` 形式の内部参照が大量にあるが、URL・書誌情報は未記載。第三者が出典を追跡するのは困難。参考文献セクションに `entity[...]` 形式で組織名が示されるが、具体的な URL・年は未記載。形式上は「出典あり」だが実質的な検証可能性は低い。

**Pro: D** — 出典・参考文献は一切なし。末尾に意味不明な 's' が残っており、出力品質管理にも若干の問題。内容は正確だが「もっともらしいが確認できない」状態。

### 5. 独自性

**ChatGPT: B+** — 「価値の束をほどく」フレームワーク（5次元 × 操作種類の二軸）が最も独自性が高い。ただしこのフレームワーク自体の出典が不明で、既存理論の援用か ChatGPT 独自の提案かは未確定。日英ズレ分析の体系性も独自。

**Pro: B** — 「四層整理（交換 / 道具的有用性 / 規範的な善さ / 生きられた重要性）」は ChatGPT の5次元と類似した分類を異なる粒度で提示。横断論点の言語化が丁寧。ただし根拠が示されていないため「GPT が作った分類」以上の重みを持たない。

**Codex: B** — 日英語彙対照（5分類 × 5語）は有用だが、百科事典の要約・再構成が主。レポート独自の仮説・モデル・図式は提示されていない。

### 6. 接続性

critic の指摘に基づき、構造的接続（セクションとして独立して論じている）と内容的接続（具体的枠組み・論点への言及）を分けて評価する。

**構造的接続**: Codex が最も高い。§5 で design / trust / AI alignment を独立セクションとして論じ、project-design の文脈に明示的に接地している。ChatGPT, Pro は領域別一覧の中に埋め込まれた言及にとどまる。

**内容的接続**: ChatGPT が最も高い。VSD（Value Sensitive Design）、CIRL（Cooperative Inverse Reinforcement Learning）を具体的枠組み名で論じている。深掘り候補にも「価値整合は倫理学の価値論を必要とするか」「価値の社会的生成 → AI 評価指標・ランキング文化への接続」を提示。

**Pro**: VSD の一言のみ。trust 言及ゼロ。project-design の中心概念（Doing/Being、Love 駆動開発）への接続は3レポートとも皆無。

### 7. 深掘り価値

**ChatGPT: A** — 10項目。各候補に citeturn 参照付き。「贈与の価値とデジタル経済」「神経現象学の方法論的射程」「価値の語彙の比較言語学（未確認と明記）」など具体的かつ学際的。次の調査を十分に駆動できる粒度。

**Pro: B+** — 10項目。ChatGPT と同数だが、優先順位や PD との関連度は示されていない。「アクティブ・インファレンスの価値語彙」「ACT の values と経済学の value の比較」など認知科学寄りの候補が充実。

**Codex: C** — 3件のみ。方向性は示されているが、具体的な問い・仮説・調査手順は提示されていない。概観に留めた結果、次の調査を駆動する力が弱い。

### 8. 簡潔さ

**Codex: A** — 160行で過不足なし。箇条書き中心で情報密度が高く、冗長な箇所はほぼない。暫定結論5項目も簡潔。

**ChatGPT: B** — 131行に情報が凝縮されているが、citeturn/entity 表記が文中に大量に埋め込まれ（1文に3-5個）、人間の可読性を著しく下げている。情報密度は高いが可読性とのトレードオフ。

**Pro: B-** — 102行と最短だが、散文形式のため情報密度にムラ。全体要約の最初2段落がほぼ同内容の反復。表や箇条書きに置き換えれば6-7割の分量で同等の情報を伝達可能。

## モデル別プロファイル

**注記**: GPT-5 / 5.2 / 5.4 のモデル世代差が出力品質にどの程度影響しているかは未分離。以下は出力結果のみに基づく評価。

### GPT-5.2 Thinking (ChatGPT)
**強み**: 構造化力（表・図の自発的使用）、網羅性（15領域）、深掘り候補の質と量
**弱み**: citeturn 形式が可読性を阻害、検証可能性が低い
**特徴**: 「束をほどく」二軸フレームワークなど独自の構造化を試みる傾向

### GPT-5.4 Pro (ChatGPT Pro)
**強み**: 横断論点の丁寧な言語化、日英翻訳ズレの一貫した扱い、認知科学寄りの深掘り候補
**弱み**: 出典ゼロ（検証可能性最低）、表・図なし（散文のみ）、出力品質管理に問題（末尾の孤立 's'）
**特徴**: 読み物としての質は高いが、調査レポートとしての証拠基盤が弱い

### GPT-5 (Codex)
**強み**: 簡潔さ、検証可能な出典（URL 付き3件）、PD 文脈への明示的接続（design/trust/alignment）
**弱み**: 網羅性が低い、深掘り候補が少ない（3件）、独自モデル・仮説の提示なし
**特徴**: 概観をコンパクトにまとめ、信頼できる出典に絞る堅実な姿勢

## project-design への示唆

### 1. 3レポートは相補的に使う

単一の最優秀レポートを選ぶより、用途に応じた使い分けが有効:

- **構造化された概観・領域マップが必要なとき** → ChatGPT
- **論点の言語化・散文的理解を深めたいとき** → Pro
- **検証可能な出典付きで簡潔な整理が必要なとき** → Codex

### 2. PD 独自の価値分類フレーム構築の可能性

ChatGPT の「5次元（価格/効用/規範/意味/象徴）× 操作（測定/正当化/解釈）」と Pro の「四層（交換/道具的/規範的/生きられた重要性）」は類似した分類を異なる粒度で行っている。両者を統合し、Doing/Being フレームと接続すれば、PD 独自の価値分類フレームを構築できる可能性がある。

### 3. 今後の調査でのモデル使い分け推奨

| 調査段階 | 推奨モデル | 理由 |
|---------|-----------|------|
| 初期概観・領域マップ作成 | ChatGPT (Thinking) | 構造化力と網羅性が高い |
| 論点の深掘り・散文的理解 | ChatGPT Pro | 横断論点の言語化が丁寧 |
| 出典付き事実確認 | Codex | URL 付き出典で検証可能 |
| PD 文脈への接続 | Codex + 手動補完 | 接続を明示するが深度は手動で補う |
| 深掘り候補の洗い出し | ChatGPT (Thinking) | 候補の質・量・具体性が最高 |

### 4. 共通の弱点

3レポートとも project-design の中心概念（Doing/Being、Love 駆動開発、欠損駆動思考）への接続はない。これはプロンプトに含まれていなかった可能性が高く、モデルの能力というよりタスク設計の問題。今後の調査では PD 固有の概念をプロンプトに含めることを推奨する。
