# 「デザイン思考とは」調査プロンプト

使用方法: このプロンプトを ChatGPT / ChatGPT Pro / Codex の3モデルに同一の形で投げる。
作成日: 2026-03-27

---

あなたは、デザイン研究・設計学・経営学・創造性研究を横断して「デザイン思考」の系譜を精密に調べる研究アシスタントです。

目的：
「デザイン思考」が、論理的思考・分析的思考との対比の中でどのように生まれ、どの時点で経営・コンサル・教育向けにパッケージ化され、どのように形骸化したのかを、一次資料・準一次資料を優先して整理してください。
特に、元来のデザイン思考が本質的にアブダクション（仮説形成・フレーミング・問題設定）に近かったのではないか、という仮説を検証したいです。

重要条件：
- Wikipedia は入口のチェックリストとして使ってよいが、権威ある根拠にはしない。
- 必ず一次資料または準一次資料（原著論文、著者本人の著作、大学・研究機関・主要実務組織の公式資料、学術レビュー）を優先する。
- 「デザイン領域の designerly thinking」と「経営・コンサル領域の design thinking」を混同しない。
- 用語のズレを厳密に区別する：
  - design thinking
  - designerly ways of knowing / designerly thinking
  - human-centered design
  - creative problem solving
  - abduction / abductive reasoning
  - deduction / induction
- 単なる賛美や流行語解説にしない。批判的系譜として書く。
- 「論理的思考」との対比は雑にしない。少なくとも deduction / induction / abduction、および analysis / synthesis / framing を分けて扱う。
- 可能なら「形骸化」が何を指すのかを具体化すること
  （例：5段階プロセスへの矮小化、共感ワークショップ化、ポストイット化、組織導入時の表層実装、評価不能な万能語化など）。
- **出典には可能な限り URL（DOI, arXiv, 公式ページ, Google Scholar リンク等）を付記すること**

調査タスク：
A. 系譜整理
- デザイン思考の起源を、設計学・デザイン研究・建築・創造研究の流れから時系列で整理する。
- その後、IDEO / d.school / Rotman などを通じて、経営・イノベーション文脈へどう移植されたかを示す。

B. 論理的思考との対比
- デザイン思考が、何に対する対抗概念として語られたのかを明示する。
- 「論理的思考」「分析的思考」「科学的方法」との違いを、 strawman にせず厳密に比較する。
- Lawson, Cross, Simon, Schon, Dorst, Martin あたりの差異も整理する。

C. アブダクションとの関係
- Peirce の abduction を起点に、design thinking がどの意味でアブダクション的なのかを整理する。
- 単に「ひらめき」ではなく、仮説形成・問題設定・リフレーミング・frame creation として説明する。
- design thinking における abduction が、科学の仮説形成とどう違うかも書く。

D. 形骸化・批判
- 学術側・実務側の双方から、design thinking への主要な批判を整理する。
- 特に、
  1) パッケージ化・商品化
  2) 5段階プロセス化
  3) 経営向けの表層化
  4) デザイン研究との切断
  5) 実証性・評価性の弱さ
  6) 組織導入時の innovation theater 化
を確認する。
- 「それでも残る本質」は何か、逆に「失われた本質」は何かを分けて書く。

E. 最終判断
次の問いに、根拠付きで答えること。
1. デザイン思考は、もともと何だったのか。
2. それは論理的思考の対立物だったのか、それとも補完物だったのか。
3. デザイン思考の核心は、アブダクション・フレーミング・問題設定にあると言えるか。
4. 今日広く流通している design thinking は、どの程度まで形骸化しているか。
5. それでもなお有効な核があるなら、それは何か。

出力形式：
- まず 400〜800字の要約
- 次に詳細版
- 最後に「結論だけ」を箇条書きで 7項目以内に要約
- 参考文献一覧を付ける
- 各主張について、できるだけ一次資料に紐づける
- 不確かな点は「未確認」「この文献からはそこまで言えない」と明記する
- **出典には可能な限り URL（DOI, arXiv, 公式ページ, Google Scholar リンク等）を付記すること**

追加要望：
- 可能なら「本来のデザイン思考」と「企業研修で流通している design thinking」を左右対比表で示す
- 可能なら「deduction / induction / abduction / design framing」の4象限比較表を作る
- 可能なら、日本語で読める重要文献や解説も最後に補足する
